## ت1 — بصمة إعدادات + إصدار الكود في الباكتيست (قياس/عرض فقط)

**الهدف:** كل رن باكتيست يحمل بصمة الإعدادات الفعّالة وإصدار الكود،
في JSON والتقرير، ويحذّر الدمج عند خلط رنّات بإعدادات مختلفة.
صفر تغيير في منطق الفرز — §6 لا ينطبق على القياس.

### 1) `backtest.py` — بناء البصمة

دالة صغيرة تُبنى من `cfg` وقت الرن:

```python
def _run_config(cfg: Config) -> dict:
    """بصمة الإعدادات المؤثّرة على نتائج الرن — للتمييز بين الرنّات."""
    return {
        "code_version": cfg.code_version or "—",
        "tech_readiness_min": cfg.tech_readiness_min,
        "entry_change_max_pct": cfg.entry_change_max_pct,
        "target1_min_rr": cfg.target1_min_rr,
        "late_wave_run_pct": cfg.late_wave_run_pct,
        "stop_fixed_pct": cfg.stop_fixed_pct,
        # + حدّا السعر بأسميهما الفعليين في Config
        "backtest_wide_t1_rr": cfg.backtest_wide_t1_rr,
    }
```

- **تحقّق من الأسماء الفعلية** للحقول في `config.py` قبل الاستخدام
  (لا تخمّن — خصوصًا حدّي السعر واسم حقل الوقف).
- لا حقول Config جديدة ولا `.env.example` — نقرأ الموجود فقط.

### 2) `save_run()` — إضافة البصمة للحمولة

في `payload` أضف `"run_config": ...`. لا تغيّر أي مفتاح قائم
(التوافق الرجعي للدمج مقدَّس — §7 بروح «لا تفترض مخططًا جديدًا»).

### 3) `format_report()` — سطر بصمة في رأس التقرير

بعد سطر «تنبيهات مُحاكاة» مباشرة، سطر واحد مثل:

```
⚙️ نسخة a1b2c3d · جاهزية≥65 · حركة≤40% · تخطّي هدف1 R/R≥0.5 · موجة متأخرة 40%
```

- `format_report` يستقبل `BacktestResult` — أضف للنتيجة حقل
  `run_config` (يُملأ في `run_backtest`) بقيمة افتراضية `None`
  كي لا تنكسر الاستدعاءات القائمة.
- لو `run_config is None` (رن قديم/دمج مختلط) → اعرض
  `⚙️ إعدادات الرن: غير موقّعة (رن قديم)`.
- لو `entry_change_max_pct <= 0` اعرض «حركة: بلا سقف» (بوّابة معطّلة).

### 4) `merge_saved_runs()` — تحذير خلط الإعدادات

- اجمع `run_config` من كل ملف (يغيب في الملفات القديمة → بصمة
  مستقلة باسم «غير موقّع»).
- لو وُجد أكثر من بصمة مختلفة (قارن الـdict كاملًا) → أضف لملاحظات
  الدمج: `⚠️ الرنّات المدموجة بإعدادات مختلفة (X بصمة) — قارن الشرائح بحذر`.
- النتيجة المدموجة: `run_config` يُملأ فقط لو كل البصمات متطابقة،
  وإلا `None` (فيظهر «غير موقّعة» — صادق مع الواقع).
- **لا ترفض الدمج** — حذّر فقط (البوت دليل لا منفّذ).

### 5) الاختبارات (`tests/test_backtest*.py`)

بمحاكاة، بلا شبكة (اقرأ الاختبارات القائمة وطابِق نمطها):
1. `save_run` يكتب `run_config` وفيه `tech_readiness_min` الصحيح.
2. `format_report` يعرض سطر البصمة عند وجودها، وسطر «غير موقّعة»
   عند غيابها.
3. دمج ملفّين ببصمتين مختلفتين → التحذير يظهر و`run_config` المدموج None.
4. دمج ملف قديم (بلا `run_config`) لا يكسر شيئًا (توافق رجعي).

### 6) القبول

- كل الاختبارات خضراء: `python -m pytest runner_scanner/tests/ -q`.
- diff جراحي: `backtest.py` + اختبارات فقط. لا مساس بأي عتبة فرز.
- نظّف pycache، commit عربي يشرح ماذا ولماذا، ثم push للفرع.

## خارج النطاق — لا تنفّذها مهما بدت مغرية

فَحص المشرف الشرائح المجمّعة على رن التحقّق (218 صفقة) ولم تتأهّل
أي منها لتعديل:

- **الوقف المتعقّب**: يبدو أفضل مجمّعًا (+4.01% مقابل +2.91%) لكن
  الحافة كلها من صفقة يناير واحدة (TCGL +182%) — **بدون يناير الدلتا
  −0.01% (صفر)**. ذيل وحيد لا تُبنى عليه سياسة خروج. القرار السابق
  «لا تغيير خروج» يثبت للمرة الثانية.
- **توسيع هدف1 للبقية R/R<0.5** (n=19): الظل يقول +1.09%→+3.21% لكن
  العيّنة صغيرة وهذي صفقات لا مقاومة أبعد مؤهّلة لها أصلًا — يبقى قياسًا.
- **بوّابة الخبر السلبي**: n=7 — عيّنة لا تُبنى عليها بوّابة.
- **أفترهاوس**: n=7 نجاح 50% — الوسم التحذيري القائم (م4) كافٍ ومطابق.
- **تحت MA200**: +3.05% (موجبة تمامًا هذي المرّة) — لا بوّابة.
- **أي تعديل عتبة فرز** — قرار المستخدم بالبيانات (CLAUDE.md §6).
  النظام مُتحقَّق؛ الحكم النهائي الآن للفيد الحي عبر `/improve`.
