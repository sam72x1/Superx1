# 🚀 ماسح الرَنرات اللحظي (Real-Time Runner Scanner)

بوت تيليجرام بايثون يراقب **ناسداك + نيويورك** لحظيًا. أول ما سهم يكسر
**+20% عن إغلاق أمس**، يلتقطه ويحلّله فورًا، ويرسل بطاقة تيليجرام فيها:
**⛔ الوقف · 🎯 الأهداف · 💪 درجة من 100**.

> **المبدأ المحوري:** سهم ممكن يكون زخمه اللحظي قوي جدًا (+20%، RVol عالي)
> **لكنه ينهار** لأنه **مو جاهز فنيًا**. فالقرار = **زخم لحظي + جاهزية فنية
> كلاسيكية معًا**. لازم الاثنتان، لا واحدة.

البوت **دليل لا منفّذ**: ينبّه فقط؛ المستخدم يقرّر ويضع الصفقة بنفسه.

---

## 🧠 كيف يحلّل (الركيزتان)

```
1) بوابات صارمة      → فشل أي وحدة = خارج تمامًا (لا درجة)
2) ركيزة الزخم /50    → RVol(حسب الجلسة) · 5min Δ% · 5min RVol · موقع VWAP · الحجم
3) ركيزة الجاهزية /50 → تحليل كلاسيكي (اتجاه/متوسطات/RSI/MACD/دايفرجنس/حجم) D/W/M
4) الدرجة النهائية   = زخم + جاهزية + تقوية الخبر، بشرط:
                       • الجاهزية الفنية ≥ 70/100
                       • الزخم فوق الحد الأدنى
```

**البوابات الصارمة:** فلوت ≤ 20M · RVol ≥ 5x · حجم ≥ 300K · سعر $1–$30
(يستثني السنتات) · رفض البارابولِك المنهك · +20% (شرط ضروري غير كافٍ).

**التركيز:** كل دورة نحلّل **أعلى 15 سهم صعودًا فقط** (`TOP_N_RUNNERS`) —
الأقوى أولًا، لا نُغرق التحليل.

**الخبر/المحفّز = إشارة تقوية لا بوّابة:** وجود خبر حديث يرفع الدرجة ويوسم
«محفّز ✓»، وغيابه لا يرفض السهم (أخبار Massive مصدرها Benzinga، مو شاملة
لرَنرات الـ small-cap).

## 🔬 أداة التطوير (Dev Assistant) — تتعلّم من نتائج البوت

البوت **يتابع كل تنبيه** بعد إرساله (من نفس السنابشوت، بلا نداء API إضافي):
هل بلغ الرَنر هدفه (✅ نجاح) أم ضرب الوقف (🛑 خسارة)؟ ويسجّل كل مرشّح
(حتى المرفوض) لاكتشاف **الفرص الفائتة**.

`dev_assistant.py` يحلّل هذا المتراكم ويُنتج تقريرًا:
- نسبة النجاح **بالشرائح** (جلسة · فلوت · RVol · 5min RVol · الدرجة · الجاهزية · الخبر).
- 🛑 أنماط الخاسرين.
- 👻 **الفرص الفائتة** (مرفوض صعد ≥30%) مجمّعة حسب **سبب الرفض** → أي بوّابة متشدّدة.
- 💡 اقتراحات ضبط للبوابات (اقتراح فقط — لا يغيّر إعدادات).

يُرسَل **تلقائيًا عند إغلاق السوق** كل يوم، أو يدويًا:

```bash
python -m runner_scanner.dev_assistant
```

## 📡 مصدر البيانات: Massive (= Polygon.io)

`massive.com` هو الاسم الجديد لـ Polygon.io بعد rebrand رسمي (30 أكتوبر
2025). نفس المفتاح يعمل على `api.massive.com` و `api.polygon.io`.

| الاستخدام | الـ endpoint |
|-----------|-------------|
| كشف +20% | `GET /v2/snapshot/locale/us/markets/stocks/tickers` (كل السوق بنداء واحد) |
| شموع 5د/1د/يومي | `GET /v2/aggs/ticker/{t}/range/...` |
| الفلوت | `GET /stocks/vX/float` (تجريبي) + fallback `share_class_shares_outstanding` |
| الخبر | `GET /v2/reference/news` |
| **التوقّفات LULD** | **WebSocket فقط** — قناة `LULD.*` + `T.*` (لا يوجد REST للتوقّف) |

> ⚠️ **اشتراكك لازم يكون لحظيًا فعلًا** (non-professional + إقرار SIP).
> بدونه = تأخير 15 دقيقة يُفسد الكشف اللحظي. الخطة الموصى بها:
> **Stocks Advanced ($199/شهر)**.

## ⚙️ التشغيل المحلي

```bash
pip install -r requirements.txt
cp .env.example .env          # ثم املأ المفاتيح الثلاثة

# افحص أن المفاتيح والاتصالات تشتغل (يرسل رسالة تجريبية لتيليجرام):

# تجربة المسح بلا إرسال:
DRY_RUN=true python -m runner_scanner.main

# التشغيل الفعلي:
python -m runner_scanner.main
```

## ☁️ النشر على Render

Background Worker مستقل **+ قرص دائم** (إلزامي لمنع تكرار التنبيه عبر
إعادة النشر — درس CCXI). الملف `render.yaml` جاهز:

1. اربط الريبو في Render → New → Blueprint.
2. أدخل الأسرار في لوحة Render: `MASSIVE_API_KEY` · `TELEGRAM_BOT_TOKEN` ·
   `TELEGRAM_CHAT_ID`.
3. تأكّد أن `DB_PATH` يطابق `mountPath` القرص (`/var/data`).

> تجنّب الطبقات المجانية اللي تستعيد الخامل (تموت بصمت وسط الجلسة).

```

  main.py          حلقة REST + keep-alive + ثريد التوقّفات + إيقاف نظيف
  pipeline.py      خط معالجة مرشّح واحد (قابل للاختبار)
  config.py        كل العتبات (env-overridable)
  models.py        نماذج البيانات المشتركة (dataclasses)
  massive_client.py غلاف REST لـ Massive
  sessions.py      كشف الجلسة + RVol حسب الجلسة
  detector.py      كشف عبور +20% من السنابشوت
  gates.py         البوابات الصارمة
  indicators.py    مؤشرات بايثون نقية (SMA/EMA/RSI/MACD/VWAP/...)
  intraday_ta.py   ركيزة الزخم اللحظي /50
  classic_ta.py    ركيزة الجاهزية الفنية الكلاسيكية 0–100
  catalyst.py      الخبر/المحفّز (إشارة تقوية)
  halts.py         آلة حالة LULD/T12 مغذّاة بـ WebSocket
  risk.py          الوقف الهجين + الأهداف
  scoring.py       دمج الركيزتين + الشروط
  alerts.py        بطاقة تيليجرام + ترتيب الأولوية + الإرسال
  state.py         SQLite: منع تكرار + closed-loop
  monitor.py       مراقبة صامتة 🚨 فقط عند عطل
  tests/           اختبارات بلا إنترنت (محاكاة ردود Massive)
```

## 📊 جلسات السوق

البوت يشتغل **بريماركت (بعد 4:00ص ET) + رسمي + أفترهاوس**:

- **RVol يُحسب حسب الجلسة** (حجم البريماركت مقابل متوسط بريماركت، لا اليومي).
- البطاقة توسم الجلسة، وتحذّر أن **LULD لا يحمي خارج الجلسة الرسمية**.
- السنابشوت يُمسح 3:30–4:00ص ET → المسح يبدأ بعد 4:00ص.
